# 🍷 End-to-End MLOps Pipeline for Wine Quality Prediction

> **MLSD Course Project** — Machine Learning System Design  
> A clean, reproducible regression pipeline built with industry-standard MLOps tools.

---

## 📌 Problem Statement

Wine quality assessment is traditionally performed by human experts, which is both subjective and expensive. This project builds a machine learning regression model that **predicts the quality score of red wine** (on a scale of 0–10) based on 11 measurable physicochemical properties. The entire workflow — from data versioning to model serving — is automated using modern MLOps practices.

---

## 📊 Dataset

| Property | Details |
|---|---|
| **Name** | Wine Quality — Red Wine |
| **Source** | [UCI Machine Learning Repository](https://archive.ics.uci.edu/ml/machine-learning-databases/wine-quality/) |
| **File** | `winequality-red.csv` |
| **Rows** | 1,599 |
| **Features** | 11 physicochemical inputs |
| **Target** | `quality` (integer score 3–8) |
| **Task** | Regression |

### Features

| Feature | Description |
|---|---|
| fixed acidity | Most acids that do not evaporate readily |
| volatile acidity | Amount of acetic acid (vinegar taste at high levels) |
| citric acid | Adds freshness and flavour |
| residual sugar | Sugar remaining after fermentation |
| chlorides | Amount of salt |
| free sulfur dioxide | Free form of SO₂ (prevents microbial growth) |
| total sulfur dioxide | Total SO₂ (free + bound) |
| density | Density relative to water |
| pH | Acidity level (0–14 scale) |
| sulphates | Additive contributing to SO₂ |
| alcohol | Alcohol percentage by volume |

## 🤖 ML Models Used

| Model | Why Used |
|---|---|
| `LinearRegression` | Baseline — checks if linear relationships exist |
| `DecisionTreeRegressor` | Captures non-linear patterns, interpretable |
| `RandomForestRegressor` | Ensemble — reduces overfitting, typically best performer |
| `GradientBoostingRegressor` | Sequential boosting — often achieves high accuracy |

The best model (highest R² score) is automatically selected and saved.

---

## 📏 Evaluation Metrics

| Metric | Description |
|---|---|
| **R²** | Proportion of variance explained (higher = better, max = 1.0) |
| **MAE** | Mean Absolute Error — average prediction error in quality units |
| **RMSE** | Root Mean Squared Error — penalises large errors more |

## 🚀 Local Setup (Windows CMD / PowerShell)

### 1. Clone the repository

```powershell
git clone https://github.com/YOUR_USERNAME/mlops-wine-quality.git
cd mlops-wine-quality
```

### 2. Create and activate virtual environment

```powershell
python -m venv venv
venv\Scripts\activate
```

### 3. Install dependencies

```powershell
pip install -r requirements.txt
```

### 4. Train the model

```powershell
python src/train.py
```

Expected output:
- `models/wine_model.pkl` — saved best model
- `reports/metrics.json` — evaluation metrics
- `reports/model_comparison.png` — bar chart
- `reports/actual_vs_predicted.png` — scatter plot

### 5. Run the Streamlit app

```powershell
streamlit run app.py
```

Open your browser at: **http://localhost:8501**

### 6. View MLflow experiment dashboard

```powershell
mlflow ui
```

Open your browser at: **http://localhost:5000**

## 📦 DVC Setup (Data Version Control with AWS S3)

```powershell
# Initialise Git and DVC
git init
dvc init

# Track the dataset with DVC
dvc add data/winequality-red.csv

# Add a remote S3 bucket for storage
dvc remote add -d s3remote s3://YOUR_BUCKET_NAME/wine-quality

# Push data to S3
dvc push

# Commit DVC tracking files to Git
git add data/winequality-red.csv.dvc .dvc/config .gitignore
git commit -m "chore: add DVC tracking for dataset"
git push origin main
```

> **To reproduce the full pipeline later:**
> ```powershell
> dvc pull       # pulls data from S3
> dvc repro      # reruns train.py if deps changed
> ```

## ⚙️ GitHub Actions CI/CD

The workflow file at [`.github/workflows/ml.yml`](.github/workflows/ml.yml) automatically runs on every push to `main`.

### Pipeline steps:

1. ✅ Checkout repository
2. ✅ Set up Python 3.10
3. ✅ Install dependencies
4. ✅ Configure AWS credentials (from GitHub Secrets)
5. ✅ `dvc pull` — fetch dataset from S3
6. ✅ `dvc repro` — retrain model if dependencies changed
7. ✅ Print `reports/metrics.json` in CI logs
8. ✅ Login to DockerHub
9. ✅ Build Docker image
10. ✅ Push image to DockerHub

### Required GitHub Secrets

Go to: **GitHub Repo → Settings → Secrets → Actions**

| Secret Name | Description |
|---|---|
| `AWS_ACCESS_KEY_ID` | AWS IAM user access key |
| `AWS_SECRET_ACCESS_KEY` | AWS IAM user secret key |
| `AWS_DEFAULT_REGION` | e.g. `us-east-1` |
| `DOCKER_USERNAME` | Your DockerHub username |
| `DOCKER_PASSWORD` | Your DockerHub access token |
